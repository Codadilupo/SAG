Enhanced XML Parsing Screen
 
Edit Enhanced XML Parsing Screen
Screen Heading: Settings > Enhanced XML Parsing
Use the Enhanced XML Parsing screen to view the default configuration that Integration Server uses for parsing an XML document using the enhanced XML parser. You can also indicate whether or not the enhanced XML parser uses cache and/or BigMemory.
To open the Enhanced XML Parsing screen, click Enhanced XML Parsing under the Settings menu on the Navigation panel.
Click Edit Enhanced XML Parsing Settings to change the enhanced XML parsing settings for this server.
Enhanced Parser Settings
Default partition size
The size, measured in bytes, of the partitions on the heap where the enhanced XML parser stores parsed document information.
Caching
Enabled
Indicates if Integration Server makes use of cache with the enhanced XML parser.
Yes indicates that during parsing of an XML document, Integration Server moves partitions to an off-heap area on disk when the on-heap partition space becomes full and retrieves the partitions as needed during processing.
No indicates that Integration Server does not cache partitions when parsing an XML document.
Maximum heap allocation for all documents combined
Maximum amount of heap space that the parser can allocate to process documents concurrently.
Maximum heap allocation for any single document
Maximum amount of heap space, that the parser can allocate to process a single document.
BigMemory Caching
Enabled
Indicates if BigMemory is used with the enhanced XML parser.
Yes indicates that when parsing an XML document using the enhanced XML parser Integration Server moves partitions to BigMemory once the on-heap cache is full. When BigMemory becomes full partitions are move to disk. Integration Server retrieves partitions from disk or BigMemory as needed.
No indicates that when parsing an XML document using the enhanced XML parser Integration Server does not use BigMemory to store cached partitions.
Maximum BigMemory allocation
Maximum amount of non-heap memory to allocate to BigMemory.
Peak Usage Statistics
Peak heap allocation for all documents combined
The maximum amount of heap used by the enhanced XML parser for parsing multiple documents at one time.
Peak heap allocation for any single document
Maximum amount of heap used by the enhanced XML parser for parsing one document.
Peak BigMemory use
Maximum amount of BigMemory used by the enhanced XML parser at one time.
