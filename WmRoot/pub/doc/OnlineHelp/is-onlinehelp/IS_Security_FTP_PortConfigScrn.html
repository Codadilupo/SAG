Edit FTP Port Configuration Screen
Screen Heading: Security > Ports > Edit FTP Port Configuration
Use the Edit FTP Port Configuration screen to view or specify the settings for an FTP port.
To open the Edit FTP Port Configuration screen, go to the Add Port Screen and select webMethods /FTP. Click Submit.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Note:  
You cannot edit port settings for the primary port.
Enable
Controls whether or not the port is enabled. When the port is enabled, it can receive requests.
Note:  
A disabled port is not re-enabled by restarting the server. A disabled port does not accept requests until you explicitly enable it.
Port
Specifies the number of the port to which you want to assign an FTP listener. Type a port number that is not used by another port on your server.
Alias
An alias for the port. An alias must include one or more of the following: ASCII characters, numbers, underscore (_), period (.), and hyphen (-).
Description
A description of the port.
Package Name
Specifies the name of the package that you want to associate with this port. When you assign a package to a port, the server enables and disables the port in conjunction with the package--if you disable the port, the server disables the port automatically. A package that you assign to a port also retains its port assignment when it is published to other servers. If a subscriber installs the package, an FTP port with this number and the same settings is created automatically on the subscriber's server. If you do not have a package to assign to the port, leave the Package Name setting at WmRoot (the server's core package). This makes the port available whenever the server is running.
Bind Address
Optional. IP address to which to bind this port. Specify a bind address if your machine has multiple IP addresses and you want the port to use this specific address. If you do not specify a bind address, the server picks one for you.
Passive Mode Listen Address
Optional. Specifies the address that should be sent by the PORT command. A host name or IP address can be specified.
Note:  
This option is not applicable when the FTP port is bound to an IPv6 address. In that case, the passive mode listen address is the same as the port bind address.�
When running in passive mode, the FTP port sends a PORT command to the FTP client. The PORT command specifies the address and port to which the client should connect to create a data connection. If the FTP port is behind a NAT server, however, the address of the host on which the Integration Server runs is not visible to the FTP client. Consequently the PORT command does not contain the information the client needs to connect to the server. To remedy this situation, you can specify a value for the watt.net.ftpPassiveLocalAddr property in the server configuration file (server.cnf), which is located in the Integration Server_directory \instances\instance_name\config directory.
Alternatively, you can use the Passive Mode Listen Address field to specify the passive mode address for an individual FTP port. That way, you can specify a different passive mode address for each FTP port. If an address is specified in the Passive Mode Listen Address field and in the watt.net.ftpPassiveLocalAddr property, the PORT command uses the value specified in the watt.net.ftpPassiveLocalAddr property.
Save Changes
Updates the server with the new or changed port settings. Your changes take effect immediately.
