Universal Messaging Connection Alias Details Screen
Screen Heading: Settings > Messaging > webMethods Messaging Settings > Universal Messaging Connection Alias
To open the Universal Messaging Connection Alias Details screen, click Messaging under the Settings menu on the Navigation panel. On the Messaging Screen, click webMethods Messaging Settings.
From the webMethods Messaging Settings Screen, in the Connection Alias Name column, click the Universal Messaging connection alias whose details you want to view.
Click Return to webMethods Messaging Settings to return to the previous screen.
Click Edit Universal Messaging Connection Alias to open the Edit Universal Messaging Connection Alias Screen and edit the Universal Messaging connection alias details.
General Settings
Connection Alias Name
Name of the messaging connection alias.
Description
Description of the messaging connection alias.
Client Prefix
A string that identifies the Integration Server to Universal Messaging. If your Integration Server belongs to a cluster, make sure all servers in the cluster use the same client prefix.
Share Client Prefix
When selected, indicates that Integration Server shares the client prefix with multiple Integration Servers and you want to prevent automatic updates, including deletions, to shared objects associated with this alias on Universal Messagingserver.
Connection Settings
Realm URL
The URL for the Universal Messaging server in the format nsp:// UM_host: UM_port (for example, nsp://127.0.0.1:9000 or nsp://localhost:9000).
Maximum Reconnection Attempts
Specify the maximum number of reconnection attempts that Integration Server makes if the connection to Universal Messaging fails. If the connection cannot be re-established, Integration Server writes messages to the error log and the connection created by this messaging connection alias will be stopped. The default is 5 attempts.
Producer Settings
Publish Wait Time while Reconnecting (milliseconds)
Number of milliseconds that a publishing service using this Universal Messaging connection alias will wait for a connection to the Universal Messaging server to be re-established after the connection fails. If Integration Server re-establishes the connection before the Publish Wait Time While Reconnecting elapses, the publishing service continues executing. If the specified time elapses before a connection is re-established, the publishing service ends with an ISRuntimeException. The default is 0 milliseconds, which means that publishing services will not wait for Integration Server to re-establish the connection.
Client Authentication Settings
Client Authentication
Type of authentication that an Integration Server client will use to connect to a Universal Messaging server. None indicates that no authentication will be performed. Username/Password indicates that basic client authentication will be performed using user name/password combinations. SSL indicates that Integration Server uses a secure socket layer (SSL) to connect to Universal Messaging.
Truststore Alias
The alias for the truststore that contains the certificate authority (CA) certificates for the Universal Messaging server. You must select a truststore alias if the Universal Messaging port is configured for either one-way authentication or two-way authentication.
Keystore Alias
The alias for the keystore that contains the client certificates that you want Integration Server to use when connecting to the Universal Messaging port. You must select a keystore alias if the Universal Messaging port is configured for two-way authentication.
Key Alias
The alias to the key that contains the private key for connecting to the Universal Messaging port securely. The key alias must be in the keystore specified in Keystore Alias.
