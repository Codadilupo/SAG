Edit State for Individual JMS Triggers Screen
Screen Heading: Settings > Messaging > JMS Trigger Management > Edit State > triggerName
Use the Edit State for Individual JMS Triggers screen to disable, suspend, or resume specific JMS triggers.
To open the Edit State for All JMS Triggers screen, click Messaging under the Settings menu on the Navigation panel. Under JMS Configuration on the Messaging Screen, select JMS Trigger Management. In the Individual JMS Trigger Controls section, click the contents of the Enabled column for the row containing the JMS trigger you wan to edit. The Edit State screen for that particular JMS trigger will be displayed.
Click Return to JMS Trigger Management to return to the previous screen.
Edit Trigger State
Use this section to change the state of a specific JMS trigger.
New State
Specifies the state for the JMS trigger.
Select...
To...
Enabled
The JMS trigger is running and connected to the JMS provider. Integration Server retrieves and processes messages for the JMS trigger.
Disabled
The JMS trigger is stopped. Integration Server neither retrieves nor processes messages for the JMS trigger.
Suspended
The JMS trigger is running and connected to the JMS provider. Integration Server has stopped message retrieval, but continues processing any messages it has already retrieved.
Save Changes
Saves the specified state change. Server restart is not required for these changes to take effect. Any changes you make are persisted across server restarts.
