Screen Heading: Server > Statistics
Use the Server Statistics screen to monitor activity on the server.
The Server Statistics screen is the default view when you open the Integration Server Administrator, or click Statistics under the Server menu on the Navigation panel.
Usage
The fields in this section display information about session and thread activity.
Total Sessions
Displays statistics for session activity.
This field...
Displays...
Current
Displays the number of sessions currently active. Click this number to display the Current Sessions Screen, containing detailed information about the active sessions.
Peak
Displays the highest number of sessions that have ever run concurrently on the server.
Limit
Not applicable to this measurement.
Licensed Sessions
Displays statistics for licensed session activity (excludes sessions used by the Administrator).
This field...
Displays...
Current
Displays the number of licensed sessions currently active.
Peak
Displays the highest number of licensed sessions that have ever run concurrently on the server.
Limit
Displays the maximum number of sessions for which the server is licensed.
Stateful Sessions
Displays statistics about stateful sessions on the server.
This field...
Displays...
Current
Displays the number of stateless sessions currently active. Click this number to display the Current Sessions Screen, containing detailed information about the stateful sessions that are currently active.
Peak
Displays the highest number of stateful sessions that have ever run concurrently on the server.
Limit
Displays the maximum number of concurrent stateful sessions on the server.
Service Instances
Displays statistics about running services.
This field...
Displays...
Current
Displays the number of running services currently active.
Note:  
This count reflects both top-level services and any child services they may call. This count does not include internal server services that are running from the wm.server. folder of the WmRoot package.
Peak
Displays the highest number of running services that have ever been active concurrently on the server.
Note:  
This count reflects both top-level services and any child services they may call. This count does not include internal server services that are running from the wm.server. folder of the WmRoot package.
Limit
Not applicable to this measurement.
Service Threads
Displays statistics about the threads spawned by the server’s running services.
This field...
Displays...
Current
Displays the number of service threads currently active.
Note:  
This count reflects top-level services only. Child services use the same thread that their top-level service uses. This count does not include services that are running from the wm.server. folder of the WmRoot package.
Peak
Displays the highest number of service threads that have ever been active concurrently on the server.
Note:  
This count reflects top-level services only. Child services use the same thread that their top-level service uses. This count does not include services that are running from the wm.server. folder of the WmRoot package.
Limit
Not applicable to this measurement.
System Threads
Displays statistics about threads spawned by the server’s internal processes (e.g., the Cache Manager, the Job Manager, the Event Handler).
This field...
Displays...
Current
Displays the number of system threads currently active. Click this number to display the System Threads Screen, containing detailed information about the threads that are currently active.
Peak
Displays the highest number of system threads that have ever been active concurrently on the server.
Limit
Not applicable to this measurement.
Uptime
Displays the length of time (hh:mm:ss) that the server has been running since it was last started.
Requests
The fields in this section display statistics about requests that this server is processing. (A request represents a service request from an external client.)
Completed Req's
Displays statistics about the number of requests that have been completed by the server.
This field...
Displays...
Current
Displays the total number of requests that were completed during the polling interval (the last 10 minutes on most servers).
Lifetime
Displays the total number of requests that were completed since the server was started.
Average Time
Displays the average time to complete a request.
This field...
Displays...
Current
Displays the average for the poll interval (the last 10 minutes on most servers).
Lifetime
Displays the average for the time period since the server was started.
Service Errors
Displays statistics about requests that ended unsuccessfully (i.e., ended with an error in the pipeline).
This field...
Displays...
Current
Displays the total number of requests that have ended unsuccessfully during the current log interval (since the log file was rotated, which is every 24 hours on most servers). Click this number to display the Error Log.
Lifetime
Not applicable to this measurement.
Req per minute
Displays the average number of requests per minute.
This field...
Displays...
Started
Displays the average number of requests that started each minute during the current polling interval (the last 10 minutes on most servers).
Ended
Displays the average number of requests that ended each minute during the current polling interval.
Memory
The fields in this section describe the availability of memory on the server. The graph shows the memory usage over the last 24 hours. The bars represent the average usage by hour as a percentage of total memory.
Maximum
Displays the maximum amount of memory the Java virtual machine (JVM) is allowed to acquire, as determined by the wrapper.java.maxmemory Java property specified in the custom_wrapper.conf file. Because the JVM reserves some survivor space for garbage collection, the displayed value depends on the underlying garbage collection algorithm used. Consequently, the displayed value might be less than the value set for the wrapper.java.maxmemory property.
Committed
Displays the amount of memory guaranteed to be available for use by the Java virtual machine.
Used
Displays the amount of memory currently used by the Java virtual machine.
Free
Displays the amount of unused memory that is currently available to the Java virtual machine.
