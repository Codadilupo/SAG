Browse Folders Screen
Screen Heading: Packages > Management > Folders > Top Level
Use the Browse Folders screen to examine the contents of a folder.
To open the Browse Folders screen, open the Packages Management Screen and click Browse Folders.
Click Return to Package Management to return to the main package management screen.
Click Up One Level to return to the screen for the parent folder (not displayed when viewing top-level folder).
Folder List
This section lists the names of folders within the folder you are viewing.
Folders
Displays the name of the folder. To examine the contents of a folder, click the folder name.
List ACL
Controls who can list the folder on screens in Designer and Integration Server Administrator. Elements contained in the folder will inherit this setting for List ACL if they have not already been assigned one. Angle brackets <> around an ACL name indicate that the folder inherits its List ACL setting from the parent folder.
Read ACL
Has no meaning for the folder itself, but elements contained in the folder will inherit this setting for Read ACL if they have not already been assigned one. Angle brackets <> around an ACL name indicate that the folder inherits its Read ACL setting from the parent folder.
Write ACL
Controls who can add, edit, and delete elements in the folder. This ACL also controls who can change ACL assignments for the folder. Elements contained in the folder will inherit this setting for Write ACL if they have not already been assigned one. Angle brackets <> around an ACL name indicate that the folder inherits its Write ACL setting from the parent folder.
Execute ACL
Has no meaning for the folder itself, but elements contained in the folder will inherit this setting for Execute ACL if they have not already been assigned one. Angle brackets <> around an ACL name indicate that the folder inherits its Execute ACL setting from the parent folder.
Folders
Displays the number of subfolders a folder contains.
Services
Displays the number of services a folder contains.
Service List
This section lists the names of services in the folder you are viewing.
Service Name
Displays the name of the service. To display detailed information about the service, click the service name.
List ACL
Controls who can list the service on screens in Designer and Integration Server Administrator. List access also allows a user to view a service's metadata. Angle brackets <> around an ACL name indicate that the service inherits its List ACL setting from the parent folder.
Read ACL
Controls who can view the main source of a service through Designer and Integration Server Administrator. Angle brackets <> around an ACL name indicate that the service inherits its Read ACL setting from the parent folder.
Write ACL
Controls who can edit the service. Angle brackets <> around an ACL name indicate that the service inherits its Write ACL setting from the parent folder.
Execute ACL
Controls who can execute the service. This access also gives the user access to files the server serves, such as DSP and .htm files. Angle brackets <> around an ACL name indicate that the service inherits its Execute ACL setting from the parent folder.
Test
Opens the Test Service Screen, which allows you to enter input values and execute the service.
