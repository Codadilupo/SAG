Create URL Alias Screen
Screen Heading: Settings > Create URL Aliases
Use the Create URL Aliases screen to create an HTTP URL alias and associate it with a service or other resource.
To open the Create URL Aliases screen, click Create URL Alias from the URL Aliases screen.
Click Return to URL Aliases to return to the URL Aliases screen.
URL Alias Properties
Alias
Name of the URL alias. The name cannot begin with the string http:// and cannot include the following characters: # % ?
There is no length restriction.
Use as empty path alias
When selected, indicates that the URL alias is associated with the empty path (/) for a hostname:port, such as localhost:5555. When you select this check box, Integration Server changes the name of the alias to <EMPTY>.
Default URL Path
The path to the resource on Integration Server. The following examples show some ways to specify URL paths:
If the resource is a service:
invoke/folder_name/service_name
If the resource is not a service:
package_name/file_name.ext
Package
Name of the package with which you want to associate this alias.
Association
Reflects where the alias was created.
Value
Description
Package
Indicates the alias was created from Integration Server Administrator.
service_name
Indicates the alias was created from Designer. When you create an alias from Designer, you can view it from Integration Server Administrator or Designer. In addition, you can move a service from one folder to another without having to update the path name assigned to the alias.
Server
Indicates that the alias was created in an earlier version of Integration Server and was migrated to this Integration Server. The alias works at this point, but the only change you can make to it is to update the package association. Once you update the package association, you can view and change the alias as you would other HTTP URL aliases.
Port Mappings
Port Number
Number of the HTTP/S port mapped to a URL path.
URL Path
The path to a resource on Integration Server.
Delete Mapping
Click to delete the mapping between the port and the URL path.
Note:  
If a URL alias specifies a default URL path and one or more port mappings, the port mappings take precedence over the default URL path.
Add Port Mapping
Port Number
Select the number of an HTTP/S port that you want to map to a URL path.
URL Path
The path to the resource on Integration Server. The following examples show some ways to specify URL paths:
If the resource is a service:
invoke/folder_name/service_name
If the resource is not a service:
package_name/file_name.ext
Add Mapping
Click to add the port mapping to this URL alias.
