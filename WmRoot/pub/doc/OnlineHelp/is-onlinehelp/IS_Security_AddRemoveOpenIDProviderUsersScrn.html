Add and Remove OpenID Provider Users Screen
Screen Heading: Security > User Management > Add and Remove OpenID Provider Users
Use the Add and Remove OpenID Provider Users screen to add, edit, or remove an OpenID Provider user. An OpenID Provider user is a type of Integration Server user that Integration Server authenticates with information received from an OpenID Provider. You can assign OpenID Provider users to groups and enable/disable OpenID Provider users as you can regular Integration Server users. However, an OpenID Provider user cannot log in to Integration Server.
To open the Add and Remove OpenID Provider Users screen, go to the User Management Screen and click Add and Remove OpenID Provider Users.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Create OpenID Provider Users
OpenID Provider User Names
Specify a user name for an OpenID Provider user. Type a unique name using any combination of letters, numbers or symbols. If you want to add more than one new user name, type each name on a separate line.
Important:  
User names on the Integration Server are case sensitive (for example, JDScott and jdscott are two distinct IDs). When you create a user name, be sure to type it exactly as you want the client to enter it.
Select OpenID Provider
Select the OpenID Provider that you want to use to authenticate the specified OpenID Provider users.
Create OpenID Provider Users
Click to create the OpenID Provider users specified in the OpenID Provider User Names field.
OpenID Provider Users
Name

OpenID Provider

Claims
