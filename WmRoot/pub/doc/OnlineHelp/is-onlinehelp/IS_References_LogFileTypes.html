Reference Information : Log File Types
Log File Types
The server maintains the following log files. Each file contains information for a particular purpose or process.
Log
File Name / Description
Error
WMERROR_yyymmdd_timestamp.log
Maintains information about all errors and exceptions that have occurred in the system. You can use this log for debugging services.
Server
server.log
Maintains information about the version of the Integration Server that is running and the operations that the server has performed. You can use this information for general debugging purposes.
Security
WMSECURITY_yyymmdd_timestamp.log
Maintains an audit trail for security-related actions, such as adding and deleting users, groups, and ACLs, changing passwords, etc.
Service
WMSERVICE_yyymmdd_timestamp.log
Maintains an audit trail for services that you configure to log data. For example, you might configure the server to add two records that contain information about the service. It adds one when it begins execution; this record includes the timestamp of when execution began. It adds the second record when the service terminates; this record includes the timestamp of when the service terminated.
Session
WMSESSION_yyymmdd_timestamp.log
Maintains information about all sessions in the Integration Server. Integration Server creates a session when a developer connects to the server from Designer or a client connects to the server to execute services. Each record in the Session Log includes a timestamp indicating when a user established a connection with the Integration Server.
Transaction
WMTXIN_yyymmdd_timestamp.log
WMTXOUT_yyymmdd_timestamp.log
Maintains information about inbound and outbound guaranteed delivery transactions. The server maintains one log for inbound and another log for outbound transactions. The server writes a record for each critical operation it performs for a guaranteed delivery transaction. Use these logs in conjunction with the appropriate job store (inbound or outbound) to determine more information about a guaranteed delivery transaction.
