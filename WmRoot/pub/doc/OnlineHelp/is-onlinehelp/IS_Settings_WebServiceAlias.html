Create Web Service Endpoints Alias Screen
Screen Heading: Settings > Web Services > Create Endpoint Alias
Use the Create Web Service Endpoints Alias screen to create a new web services alias on the Integration Server or to configure a web service endpoint alias with authentication information and X509 certificates. You can also create message addressing endpoint aliases that Integration Server uses to send responses to endpoints other than the one which initiated or sent the request. For information about web service endpoint aliases, WS-Security, and WS-Addressing, see the Web Services Developerâ€™s Guide.
To open the Web Services Alias screen, click Web Services under the Settings menu on the Navigation panel. On the Web Services Settings Screen, click Create Web Service Endpoint Alias.
Click Return to Web Service Endpoints to return to the previous screen.

Field

Applies to consumer endpoint aliases only.

Specify a text identifier for the keystore, which contains private keys and certificates for an Integration Server, partner application, or Integration Server component.

Specify a text identifier for a key in the keystore.
Truststore Alias
Specify a text identifier for the truststore that contains the list of CA certificates that Integration Server uses to validate the trust relationship.
This field applies only to consumer and provider endpoint aliases.
Timestamp Precision
Specify whether the timestamp placed in the Timestamp element of the security header of an outbound message is precise to seconds or milliseconds. If you set the precision to milliseconds, Integration Server uses the timestamp format yyyy-MM-dd'T'HH:mm:ss:SSS'Z'. If you set the precision to seconds, Integration Server uses the timestamp format yyyy-MM-dd'T'HH:mm:ss'Z'.
Integration Server uses the timestamp precision value only when you implement WS-Security via a WS-Policy.
Timestamp Time to Live (seconds)
Specify the time-to-live value for an outbound message in seconds. Integration Server uses this value to set the expiry time in the Timestamp element of outbound messages.
Integration Server uses the timestamp precision value only when you implement WS-Security via a WS-Policy.
Timestamp Maximum Skew (seconds)
Specify the maximum number of seconds that the web services client and host clocks can differ so that the timestamp expiry validation does not fail. Specify a positive integer or zero.
Integration Server uses the timestamp maximum skew value only when you implement WS-Security via a WS-Policy. Integration Server validates the inbound SOAP message only if the creation timestamp of the message is less than the sum of the timestamp maximum skew value and the current system clock time.

Reliable Messaging Properties of a Provider or Consumer Endpoint Alias (Optional)
Reliable messaging properties provide information specific to the web service endpoint. By default, web service endpoints use the reliable messaging properties defined for Integration Server on the Reliable Messaging Properties (Settings > Web Services > Reliable Messaging > Edit Configuration) page. You can override the server-level reliable messaging configuration with reliable messaging properties specific to the web service endpoint alias.
Under Reliable Messaging Properties, check Enable to provide reliable messaging information specific to the endpoint alias you are creating. Provide the following reliable-messaging information to ensure reliable delivery of the message between the two endpoints (web service and client or reliable messaging source and destination).
Field
Description
Retransmission Interval
Specify the time interval (in milliseconds) for which a reliable messaging source waits for an acknowledgement from the reliable messaging destination before retransmitting the SOAP message. The default is 6000 milliseconds.
Acknowledgement Interval
Specify the time interval (in milliseconds) for which the reliable messaging destination waits before sending an acknowledgement for a message sequence. Messages of the same sequence received within the specified acknowledgement interval are acknowledged in one batch. If there are no other messages to be sent to the acknowledgement endpoint within the specified acknowledgement interval, the acknowledgement is sent as a stand-alone message.
The default is 3000 milliseconds.
Exponential Backoff
Specify whether to use the exponential backoff algorithm to adjust the retransmission interval of unacknowledged messages. Adjusting the time interval between retransmission attempts ensures that a reliable messaging destination does not get flooded with a large number of retransmitted messages.
Select...
To...
true
Increase the successive retransmission intervals exponentially, based on the specified retransmission interval. For example, if the specified retransmission interval is 2 seconds, and the exponential backoff value is set to true, successive retransmission intervals will be 2, 4, 8, 16, 32, and so on if messages continue to be unacknowledged. This is the default.
false
Use the same time interval specified in the Retransmission Interval field for all retransmissions.
Maximum Retransmission Count
Specify the number of times the reliable messaging source must retransmit a message if an acknowledgement is not received from the reliable messaging destination. To specify that there is no limit to the number of retransmission attempts, set the value of Maximum Retransmission Count to -1. The default is 10.
