Password Security Settings Screen
Screen Heading: Security > User Management > Password Security Settings
Use the Password Security Settings screen to specify the length and character criteria for passwords. You also use this screen to specify whether users are allowed to change their own passwords.
To open the Password Security Settings screen, go to the User Management Screen and click Password Restrictions.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Edit Password Restrictions to change the password settings.
Important:  
A user who logs in with Administrator authorization can change his or her password to one that does not meet the length and character requirements.
Password Restriction Settings
Use this section to specify whether users are permitted to change their passwords and to specify the criteria that the passwords they specify must satisfy.
Enable Password Change
Specifies whether non-Administrator users are allowed to change their passwords. The default is Yes. If you select No, users who log on without Administrator authorization (users that are not members of the Administrators ACL) must have a system administrator change the passwords for them.
Minimum Password Length
Specifies the minimum number of total characters a password must contain. The default is 8.
Minimum Number of Upper Case Characters
Specifies the minimum number of uppercase alphabetic characters the password must contain. The default is 2.
Minimum Number of Lower Case Characters
Specifies the minimum number of lowercase alphabetic characters the password must contain. The default is 2.
Minimum Number of Digits
Specifies the minimum number of numeric digits the password must contain. The default is 1.
Minimum Number of Special Characters (neither alphabetic nor digits)
Specifies the minimum number of special characters, such as asterisk (*), period (.), question mark (?), and ampersand (&), the password must contain. The default is 3.
Save Password Settings
Shown only in edit mode. Saves the password settings that you have specified. The changes take effect immediately. Note that these changes do not affect passwords that already exist on the server. The new requirements will be imposed on only those passwords that users define from this point forward.
