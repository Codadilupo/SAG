Add Port Screen
Screen Heading: Security > Ports > Add Port
Use the Add Port screen to specify which type of port you want to add to your server.
To open the Add Port screen, go to the Ports Screen and click Add Port.
Click Return to Ports to return to the list of ports that are currently defined on the server. Type Specifies the type of port that you want to add. Select one of the following:
Type
Specifies the type of port that you want to add. Select one of the following:
Type of Port
Description
webMethods /HTTP
A port that accepts requests for services via Hypertext Transport Protocol (HTTP).
webMethods /HTTPS
A port that accepts requests for services via the Secure Socket Layer (SSL).
webMethods /File Polling
A port that listens for the arrival of files on a file system directory.
webMethods /FTPS
A port that accepts requests for services via secure File Transfer Protocol (FTPS).
webMethods /FTP
A port that accepts requests for services via the File Transfer Protocol (FTP).
webMethods /Email
A port that accepts requests for services via email messages from a POP or IMAP mail server.
HTTP Diagnostic
A port that uses threads from a dedicated thread pool to accept requests for services via HTTP.
HTTPS Diagnostic
A port that uses threads from a dedicated thread pool to accept requests for services via HTTPS.
Enterprise Gateway Server
In a webMethods Enterprise Gateway configuration, an Integration Server that resides in your DMZ and acts as an intermediary between the Internet and your Internal Server is called an Enterprise Gateway Server. The Enterprise Gateway Server has an external port to listen for requests from external clients (partners) and a registration port through which it maintains its connection to the Internal Server. When you select Enterprise Gateway Server, you can configure both the Enterprise Gateway external port and the Enterprise Gateway registration port. See webMethods Integration Server Administrator’s Guide for more information.
Note:  
You cannot configure an Enterprise Gateway Server port if webMethods API Gateway is installed on Integration Server. In this case, the port is disabled.
Internal Server
In a webMethods Enterprise Gateway configuration, an Integration Server that sits behind your internal firewall and is configured to receive requests from an Enterprise Gateway Server is called an Internal Server. An Internal Server maintains a persistent connection with the registration port on the Enterprise Gateway Server. When you select Internal Server, you can configure the Internal Server to connect to the Enterprise Gateway registration port. See webMethods Integration Server Administrator’s Guide for more information.
Note:  
You cannot configure an Internal Server port if webMethods API Gateway is installed on Integration Server. In this case, the port is disabled.
Submit
Opens the Port Configuration screen, which you use to configure the port for the selected protocol.
