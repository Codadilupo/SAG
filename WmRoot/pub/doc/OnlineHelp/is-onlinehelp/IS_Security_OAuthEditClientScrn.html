Edit Client Screen
Screen Heading: Security > OAuth > Client Registration > Register Client > Edit
Use the Edit Client screen to edit registered clients.
To open the Edit Client screen, go to the OAuth Screen and click Client Registration. Click the Application Name or Client ID for the client that you want to edit.
Click Return to Client Registration to return to the Client Registration Screen.
Client Configuration
ID
Client ID generated by Integration Server for the client. Integration Server requires the client ID, client secret, or both when the client invokes the OAuth services.
Note:  
You cannot edit the value for ID.
Secret
If you specified Confidential in the Type field when you registered the client, Integration Server generates a client secret for the client. Integration Server requires the client ID, client secret, or both when the client invokes the OAuth services.
Note:  
You cannot edit the value for Secret.
Name
Specifies the name of the client. The name cannot contain the following characters:
& ( ) \ ; , / " : ' < >
Note:  
You cannot create clients with the same Name and Version combination.
Version
Specifies the version number of the client. The version cannot contain the following characters:
& ( ) \ ; , / " : ' < >
Note:  
You cannot create clients with the same Name and Version combination.
Type
Specifies the type of the client according to its ability to communicate with the authorization server. Select one of the following:
Option
Description
Confidential
Indicates that the client can maintain the confidentiality of its credentials or is capable of secure client authentications. When you select client type as Confidential, Integration Server generates a client secret. This client secret will be required by Integration Server when the client makes requests to the OAuth services.
Public
Indicates that the client is not capable of maintaining confidentiality of its credentials and not capable of secure client authentications.
Description
Optional. Description of the client.
Redirect URIs
The URIs that the authorization server will use to redirect the client when the client is authorized. You can add more than one URI at a time by specifying multiple lines, one URI to a line. Press ENTER to separate lines.
Token
Expiration Interval
Specifies the length of time (in seconds) that the access token is valid. Select one of the following:
Option
Description
Use OAuth Global Settings
Indicates that Integration Server will use the setting specified in the Access token expiration interval field on the OAuth Screen. This setting is shown in angle brackets (< >). For more information about Access token expiration interval, see OAuth Screen.
Never Expires
Indicates that the access token never expires.
Expires
Specifies an exact time interval. Enter the number of seconds that the access token is valid in the field provided. The maximum value is 2147483647. The default is 3600.
Refresh Count
Specifies the number of times the access token can be refreshed. Select one of the following:
Option
Description
Unlimited
Refreshes the access token an unlimited number of times.
Limit
Integration Server refreshes the access token the specified number of times. If you specify 0 or leave the Limit field empty, Integration Server does not refresh the access token. The maximum value is 2147483647.
Save Changes
Updates Integration Server with the new or changed OAuth settings. Your changes take effect immediately.
