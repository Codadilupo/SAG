Edit State for Individual webMethods Messaging Trigger Screen
Screen Heading : Settings > Messaging > webMethods Messaging Trigger Management > Edit State > triggerName
Use the Edit State for Individual Triggers screen to suspend or resume document retrieval or document processing for a specific webMethods messaging trigger.
To open the Trigger State screen, click Messaging under the Settings menu on the Navigation panel. On the Messaging Screen, click webMethods Messaging Trigger.
In the Individual Trigger Controls section, there are headings named Document Retrieval and Document Processing, with a sub-heading named Active. To edit the state of a specific trigger, click Yes in either Active column next to the name of the trigger. The Edit State screen for that trigger will be displayed.
Click Return to Trigger Management to return to the display of trigger settings.
State
Use this section to change the document retrieval or document processing state for the selected webMethods messaging trigger.
Retrieval State
Specifies the document retrieval state for the selected trigger. Select Active to make document retrieval active for the trigger. Select Suspended to suspend document retrieval for the selected trigger.
Processing State
Specifies the document processing state for the selected trigger. Select Active to make document processing active for the selected trigger. Select Suspended to suspend document processing for the selected trigger.
Change Mode
Use this section to indicate whether state changes should be permanent or temporary and whether the Integration Server should apply the changes to all the servers in a cluster.
Apply change permanently
When selected, indicates that the state changes for the trigger are permanent and will be maintained after the Integration Server restarts, the trigger is enabled or disabled, or the package containing the trigger reloads.
Apply Change Across Cluster
When selected, indicates whether the Integration Server should apply the document retrieval and document processing state changes for this trigger to the same trigger on all the other servers in a cluster. This check box appears only if the current Integration Server belongs to a properly configured cluster and is configured to synchronize trigger changes across the cluster.
Save Changes
Saves the specified settings. Server restart is not required for these changes to take effect.
