Screen Heading: Packages > Management
Use the Package Management screen to display the names and status of packages on the webMethods Integration Server.
To open the Packages Management screen, click Management under the Packages menu on the Navigation panel.
Note:  
You must have List access to a package to be able to list it on this screen.
Click Install Inbound Releases to install packages that other webMethods Integration Servers have published to your server.
Click Activate Inactive Packages to activate inactive packages.
Click Recover Packages to recover a previously deleted package from the salvage directory.
Click Browse Folders to display the list of folders on the webMethods Integration Server.
Click View Locked Elements to display a list of user and system locked elements on the webMethods Integration Server.
Click Filter Packages to limit the packages to be displayed.
Click Show All Packages to disable filtering and restore the default list of all packages on the server. This also updates the Packages List to reflect any changes that occurred to the Integration Server (such as new packages, etc.) while Filter Packages was enabled.
Filter
Click Filter Packages to include or exclude packages that match specified criteria, and narrow down a filtered list by filtering on the results again.
Filter criteria
Specifies the string to submit to the filter. By default, packages with names that match the string are included in the results. Filter criteria can be literals or a combination of literal and wild-card characters. The "*" (asterisk) and "?" (question mark) are the only supported wild-card characters. Note that leaving the filter criteria blank includes all packages.
Important:  
The package names in the Filter criteria field are case-sensitive. If you enter "wma*", the filter will ignore any packages beginning with "WmA".
Include Enabled
Includes only packages that are enabled in the results (that is, those that have a Yes in the Enabled column of the Packages List).
Include Disabled
Includes only packages that are disabled in the results, (that is, those with a No in the Enabled column of the Packages List).
Include Both
Includes all packages in the results, whether they are enabled or disabled.
Filter on result
Limits the search to just the currently displayed list of packages (that is, the results of the previous filtering operation), rather than the original list of all packages on the server.
Exclude from result
Displays the packages that do not match the Filter criteria, rather than the packages that do match them.
Submit
Processes the filter criteria and updates the page.
Package List
This section lists the names of packages installed on this webMethods Integration Server.
Package Name
Displays the name of the package. To display detailed information about a package, click the package name.
Home
Opens the home page (if one exists) for the package, which is located in Integration Server_directory \instances\instance_name\packages\PackageName\pub\index.html. The home page is an optional page you can create to provide a user interface to the services in a package for testing, administration, or development purposes. To open the home page for a package, click the icon in this column.
Reload
Reloads the package from disk. Use this icon to reload a package that did not load successfully during startup. To reload the package specified in Package Name, click this icon.
Enabled
Indicates the availability of the package specified in Package Name. When enabled (Yes), the services in the package are available to clients. When disabled (No), access to the package is blocked. To enable or disable a package, click the No or Yes link in this field.
Note:  
When you disable a package, lock information is not retained.
Loaded
Indicates the status of the services in the package specified in Package Name.
Content
Description
Yes
All services in the package are currently loaded and available.
Partial
Some of the services in the package did not load and are unavailable. (Those services that loaded successfully are available for use.) To learn which services did not load and why, click the package name and look at the Load Errors list at the bottom of the screen.
Warnings
Some of the services in the package generated warnings, but were loaded and are available for use. To learn which services caused warnings look at the Load Warnings list at the bottom of the screen.
No
None of the services in the package are currently loaded.
Archive
Opens the Archive Package Screen, which you use to create an archive copy (a zip file) of all or part of a package. (Archive copies are stored in the Integration Server_directory \instances\instance_name\replicate\outbound directory.) To archive the package specified in Package Name, click the icon in this field.
Safe Delete
Removes the package from the webMethods Integration Server, but puts a copy in the Integration Server_directory \instances\instance_name\replicate\salvage directory so that you can recover it if necessary (see the Recover Packages Screen). To safely delete the package specified in Package Name, click the icon in this field.
Note:  
When you use Safe Delete, lock information is not retained for elements in the package.
Delete
Deletes the package specified in Package Name.
Note:  
You can delete a package on the server even if services contained in the package are locked.
