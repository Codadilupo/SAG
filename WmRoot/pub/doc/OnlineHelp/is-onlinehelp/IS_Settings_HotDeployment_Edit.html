Edit Hot Deployment Settings Screen
Screen Heading: Settings > Hot Deployment > Edit Hot Deployment Settings
Use the Edit Hot Deployment Settings screen to enable hot deployment of custom packages in Integration Server.
You use the hot deployment feature in Integration Server to seamlessly upgrade custom packages in Integration Server while ensuring that the Integration Server assets are available for processing without any noticeable downtime. For more information about hot deployment of packages, see webMethods Integration Server Administrator’s Guide.
To open the Edit Hot Deployment Settings screen, click Edit Hot Deployment Settings from the Hot Deployment Settings screen.
Click Return to Hot Deployment Settings to go back to the Hot Deployment Settings screen.
Enable
Select Yes to enable hot deployment of packages.
Timeout
Specify the maximum time in seconds Integration Server should wait for the in-flight processing to continue before attempting to hot deploy the packages. The default is 60 seconds.
Auto Recover
Select Yes, if you want Integration Server to revert to the previous version of the package in case the deployment of a new version of the package fails. When you select Yes for the Auto Recover option, Integration Server copies the original package to the Integration Server_directory \instances\instance_name\replicate\salvage directory before it deploys the new version. Thus, in case of any error during hot deployment, Integration Server recovers and reverts to the original package.
Save Changes
Saves the hot deployment settings.
