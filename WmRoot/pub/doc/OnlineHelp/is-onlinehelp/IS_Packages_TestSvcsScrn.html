Test Service Screen
Screen Heading: Packages > Management > Services > servicename > Test
Use the Test Service screen to execute the selected service as if it were invoked by an HTTP client. This screen allows you to provide input values for String-based input parameters for the test. Results are formatted using the service output template, if one is assigned, and are displayed in your browser.
To open the Test Service screen, go to the Packages Management Screen and click a package name. The Package Description Screen will be displayed. Click Browse services in packageName for a list of the services and specifications in that package. Click a service or specification name to view the detailed information for that service. Then click Test serviceName to open the Test Service Screen and execute the service.
Click Return to serviceName to return to the information screen for this service.
Assign Input Values
Use this section to specify the input values for the test. It lists the input parameters that have been declared for the service and allows you to enter test data for String parameters. Type your input values in the text boxes provided in this list.
Test (with inputs)
Executes the service with the input values specified in Assign Input Values.
Test (without inputs)
Executes the service without passing it inputs. This option is typically used to test the service's response to a "no input" condition.
