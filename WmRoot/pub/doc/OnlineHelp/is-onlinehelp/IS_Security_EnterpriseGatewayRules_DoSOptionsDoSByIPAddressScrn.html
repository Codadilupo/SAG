Configure Denial of Service by IP Address Screen
Screen Heading: Security > Enterprise Gateway Rules > Denial of Service Options > Configure Denial of Service by IP Address
Use the Configure Denial of Service by IP Address screen to restrict the number of requests Enterprise Gateway Server will:
Accept from any individual IP address.
Process concurrently from any individual address.
For example, you can specify that, if Enterprise Gateway Server receives more than 15 requests from any individual IP address in 10 seconds, subsequent requests from that IP address will be denied.
Note:  
This screen is disabled if webMethods API Gateway is installed on Integration Server.
To open the Configure Denial of Service by IP Address screen, click Enterprise Gateway Rules under the Security menu on the Navigation panel, click Denial of Service Options, then click Configure Denial of Service by IP Address.
Click Return to Denial of Service Options to return to the Denial of Service Options screen.
Enable
Whether Enterprise Gateway Server should enforce limits on the number of requests it will accept and the number of requests it will process concurrently from any individual IP address.
Maximum Requests
Maximum number of requests Enterprise Gateway Server will accept from any individual IP address in the specified time interval.
Maximum Requests in Progress
Maximum number of requests Enterprise Gateway Server will run concurrently from the same IP address.
When Limit Exceeds
Action to take if the limit specified by the Maximum Requests or Maximum Requests in Progress field is exceeded.
Select...
To...
Add to Deny List
Permanently add the IP address that exceeded the limit to the Enterprise Gateway deny list. With this setting in effect, Enterprise Gateway Server blocks requests from the IP address at every Enterprise Gateway external port. The address remains in the deny list until an administrator deletes it from the list.
Block
Temporarily add the IP address that exceeded the limit to the Enterprise Gateway deny list. With this setting in effect, Enterprise Gateway Server blocks requests from the IP address at every Enterprise Gateway external port for a period of time that you specify.
Block Interval
Length of time (in minutes) that Enterprise Gateway Server should block requests from the IP address when the number of requests exceeds either the Maximum Requests or Maximum Requests in Progress limits. The server blocks requests at every Enterprise Gateway external port.
Error Message
Custom message to send to the client, if desired, when a request is denied.
