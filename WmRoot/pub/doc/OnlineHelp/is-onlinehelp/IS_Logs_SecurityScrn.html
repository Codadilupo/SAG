Security Log Screen
Screen Heading: Logs > Security
Use the Security Log screen to view the contents of the most recent security audit log (WMSECURITY_yyyymmdd_hhmmss.log). The security audit log maintains an audit trail for all security actions that the server executes. Depending on the settings you specify for Security Configuration and Security Actions on the Settings > Logging screen, the server adds records to the log at various times, such as when security auditing was enabled or when auditing configuration was modified. See the webMethods Audit Logging Guide for more information about log display controls.
Note:  
For security purposes, you should regularly check your audit logs for security breaches. In addition, you should back them up regularly in case the machine where the logs are stored becomes unavailable.
To open the Security Log screen, click Security under the Logs menu on the Navigation panel.
Log display controls
Display Log Entries oldest to the newest starting from the beginning
View entries in the log from the oldest to the most current.
Display Log Entries newest to the oldest starting from the end
View entries in the log from the most current to the oldest.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the log. You can change the number of entries displayed.
Tip:  
If you have Administrator privileges, you can permanently change the default value of this field. Go to the Settings > Extended page and modify the watt.server.log.maxEntries parameter.
Refresh
Click this button to update the display after changing the log display controls.
Tip:  
If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Security Log Entries as of yyyy-mm-dd hh:mm:ss
Time Stamp
Date and time the entry was written to the log.
Message
Provides information about the security action.
Server Id
IP address or host name of the server on which Integration Server is running.
Client Id
IP address of the host from which the request originated.
User Id
Identity of the Integration Server user performing this security action.
Security Event Type
A specific area in Integration Server for which a security action was performed.
