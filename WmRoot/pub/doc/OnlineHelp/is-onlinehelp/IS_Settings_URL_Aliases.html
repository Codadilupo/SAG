URL Aliases Screen
 
Create URL Alias Screen
Edit URL Alias Screen
Screen Heading: Settings > URL Aliases
Use the URL Aliases screen to view HTTP URL aliases that are defined for your Integration Server.
To open the URL Aliases screen, click URL Aliases under the Settings menu on the Navigation panel.
Click Create URL Alias to create a new URL alias.
Click the underlined name of a URL alias to view or change details about a URL alias.
Alias
Name of the URL alias.
Default URL Path
The path to the service or resource on Integration Server.
Port Mapping
A mapping between an HTTP/S port and a service or resource. When Integration Server receives a request through the specified port, Integration Server routes the request to the resource mapped to the port number.
Note:  
If a URL alias specifies a default URL path and one or more port mappings, the port mappings take precedence over the default URL path.
Association
Reflects where the alias was created.
Value
Description
Package
Indicates the alias was created from Integration Server Administrator.
service_name
Indicates the alias was created from Designer. When you create an alias from Designer, you can view it from Integration Server Administrator or Designer. In addition, you can move a service from one folder to another without having to update the path name assigned to the alias.
Server
Indicates that the alias was created in an earlier version of Integration Server and was migrated to this Integration Server. The alias works at this point, but the only change you can make to it is to update the package association. Once you update the package association, you can view and change the alias as you would other HTTP URL aliases.
Package
Name of the package with which this alias is associated.
Delete
Deletes the associated URL alias.
