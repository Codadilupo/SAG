JMS Trigger Management Screen
Screen Heading: Settings > Messaging > JMS Trigger Management
Use the JMS Trigger Management screen to view global and individual controls for JMS triggers.
To open the JMS Trigger Management screen, click Messaging under the Settings menu on the Navigation panel. Under JMS Configuration on the Messaging Screen, click JMS Trigger Management.
Click Return to Messaging to return to the previous screen.
Click Edit JMS Global Trigger Controls to change the number of server threads used for JMS triggers.
Click Refresh Page to update the information on the screen.
Global JMS Trigger Controls
Use this section to view the view the maximum number of threads that can be used for JMS triggers and to view the adjusted concurrent processing level for concurrent triggers.
Thread Pool Throttle
Displays the percentage of the server thread pool that can be used for JMS triggers.
Processing Throttle
Displays the percentage of maximum allowed execution threads for concurrent JMS triggers.
Click Go to Individual SOAP JMS Trigger Controls to jump to the Individual SOAP JMS Trigger Controls table.
Individual Standard JMS Trigger Controls
Use this section to view detailed information for each standard JMS trigger on Integration Server.
Trigger Name
Fully qualified name of the JMS trigger.
Connection Alias Name
Name of the JMS connection alias that Integration Server uses to retrieve messages for the JMS trigger.
Destination(s)
The queues or topics on the JMS provider for which the JMS trigger receives messages.
Processing Mode
Message processing mode for the JMS trigger.
Processing Mode
Description
Serial
Integration Server processes messages received by the trigger one after the other.
Concurrent
Integration Server processes multiple messages for this trigger at one time.
Maximum Threads
Indicates the maximum number of server threads that can process messages for the JMS trigger concurrently. If the current maximum threads value is different from the configured maximum threads value (that is, the Execution Threads Throttle is set to less than 100%), the configured maximum threads value is displayed in parenthesis.
Current Threads
Specifies the current number of server threads processing messages for this trigger.
Enabled
Indicates whether the trigger is enabled or disabled.
edit all
Click to access the Edit State for All JMS Triggers Screen in which you can enable, disable, or suspend all the JMS triggers on the Integration Server.
Click Go to Individual Standard JMS Trigger Controls to jump to the Individual Standard JMS Trigger Controls table.
Individual SOAP JMS Trigger Controls
Use this section to view detailed information for each SOAP-JMS trigger, including WS (web service) endpoint triggers on Integration Server.
Trigger Name
Fully qualified name of the SOAP-JMS trigger. Note that WS endpoint triggers use the format WS Endpoint Trigger: alias where alias is the name of the provider web service endpoint alias that uses the WS endpoint trigger.
Connection Alias Name
Name of the JMS connection alias that Integration Server uses to retrieve messages for the SOAP-JMS trigger.
Destination(s)
The queue or topic on the JMS provider for which the SOAP-JMS trigger receives messages.
Processing Mode
Message processing mode for the SOAP-JMS trigger.
Processing Mode
Description
Serial
Integration Server processes messages received by the trigger one after the other.
Concurrent
Integration Server processes multiple messages for this trigger at one time.
Maximum Threads
Indicates the maximum number of server threads that can process messages for the JMS trigger concurrently. If the current maximum threads value is different from the configured maximum threads value (that is, the Execution Threads Throttle is set to less than 100%), the configured maximum threads value is displayed in parenthesis.
Current Threads
Specifies the current number of server threads processing messages for this SOAP-JMS trigger.
Enabled
Indicates whether the trigger is enabled or disabled.
edit all
Click to access the Edit State for All JMS Triggers Screen in which you can enable, disable, or suspend all the JMS triggers on the Integration Server.
