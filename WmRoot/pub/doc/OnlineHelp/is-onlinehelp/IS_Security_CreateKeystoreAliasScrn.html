Create a Keystore Alias
Screen Heading: Security > Keystore > Create Keystore Alias
Use the Create Keystore Alias screen to configure a new alias for an existing keystore file, which contains the private keys and certificates for an Integration Server, partner application, or Integration Server component. The alias information is saved in an XML configuration file on the Integration Server.
Note:  
The keystore file must already be configured and saved before you can supply an alias.
To open the Create Keystore Alias screen, click Keystore under the Security menu on the Navigation panel, then select Create Keystore Alias.
Click Return to Keystore to return to the list of ports that are currently defined on the server.
Keystore Properties
Alias
Specifies a text identifier for the keystore file.
Description
Specifies a text description for the keystore alias.
Type
Specifies the file format of the keystore file. Integration Server supports JKS and PKCS12 file formats for keystores.The default is JKS. Other keystore types can be made available by loading additional security providers, or by setting the watt.security.keyStoreTypes server configuration property.
Provider
Specifies the provider that is used for the keystore. The default is the provider shipped with the JVM (Oracle, IBM, or others). If your Hardware Security Module (HSM) device is not supported by the default provider, you can configure a different provider to support the keystore filetype.
Location
Specifies the location of the keystore file on the server. This can be the full pathname or a relative path in relation to the Integration Server.
Password/Re-type Password
Specifies the password for the saved keystore file associated with this alias. This password must have been defined at the time the keystore was created, using your keystore utility. Make sure that you have this password available when managing keystore aliases, as you cannot access it through the Integration Server.
HSM Based Keystore
Specifies whether the keystore is stored on a Hardware Security Module (HSM) device. Only nCipher hardware card modules are currently supported. If True is selected, no path is specified in the Location field.
Submit
Click to create the alias.
These fields are displayed after you click Submit to create the alias, if you have a valid keystore specified.
Alias
Lists each alias found in the keystore.
Password/Re-type Password
Specifies a password for each alias found in the keystore. Most aliases require a password. If Integration Server needs to use this alias for any reason, you must provide its password.
Null
Specifies that the alias is not secured with a password.
Save Changes
Click to submit the password entries for the keystore aliases.
