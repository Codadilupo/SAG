Hot Deployment Settings Screen
 
Edit Hot Deployment Settings Screen
Screen Heading: Settings > Hot Deployment
The Hot Deployment Settings screen displays the settings for the hot deployment of packages in Integration Server.
You use the hot deployment feature in Integration Server to seamlessly upgrade custom packages in Integration Server while ensuring that the Integration Server assets are available for processing without any noticeable downtime. For more information about hot deployment of packages, see webMethods Integration Server Administrator’s Guide.
To open the Hot Deployment Settings screen, click Hot Deployment under the Settings menu on the Navigation panel.
Click Edit Hot Deployment Settings to specify or edit the hot deployment settings.
Enable
Specifies whether hot deployment of packages is enabled. Yes indicates that hot deployment of custom packages is enabled. No indicates that hot deployment is not enabled. Hot deployment is disabled by default.
Timeout
Specifies the maximum time in seconds Integration Server should wait for the in-flight processing to continue before attempting to hot deploy the packages. The default is 60 seconds.
Auto Recover
Specifies whether Integration Server reverts to the previous version of a package in case the deployment of a new version of the package fails. Yes indicates that Integration Server reverts to the previous version of the package in case a hot deployment failure. Integration Server copies the original package to the Integration Server_directory \instances\instance_name\replicate\salvage directory before it deploys the new version. Thus, in case of any error during hot deployment, Integration Server recovers and reverts to the original package. No indicates that Integration Server will not revert to the previous version of the package in case the deployment of a new version of the package fails. Auto recovery is disabled by default.
